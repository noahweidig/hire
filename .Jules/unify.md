## 2026-03-08 - Narrative Flow: Credibility Before Skills
**Learning:** Placing actual examples of work (proof/credibility) before abstract lists of skills creates a stronger, more logical narrative flow. Users see the results first, making the skills listed later more meaningful.
**Action:** When evaluating 'hire me' or portfolio pages, always check if tangible proof of capability is presented before long lists of tools or skills. Reorder them if necessary to build credibility earlier in the scroll.

## 2026-03-08 - Value proposition precedes proof
**Learning:** The "Why Hire Me" section was buried beneath a long block of skills, creating a disconnected narrative. Visitors were asked to review extensive technical capabilities before understanding the value proposition and core differentiators. Moving value proposition immediately after the primary introduction ("What I Do") establishes "why this matters" sooner, creating a stronger foundation before presenting raw skills or proof.
**Action:** When evaluating single-page portfolios, always ensure the narrative flow builds logically: Hero -> Services/Offerings -> Value Proposition -> Proof/Work -> Specific Skills/Capabilities. Reorder sections and ensure the navigation map mirrors this logical arc.
